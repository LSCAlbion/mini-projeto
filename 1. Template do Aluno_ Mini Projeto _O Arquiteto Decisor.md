### 1.1 Resumo do Cenário de Negócio

O sistema HealthSync é uma plataforma de gestão de saúde pública focada no combate a endemias através de inteligência artificial. O principal problema que a arquitetura visa resolver é a otimização das rotas dos agentes de saúde, prevendo áreas de maior risco de infestação e estimando a quantidade ideal de larvicida para o percurso. O sistema entrega valor ao economizar tempo e esforço do agente na rua, guiando-o diretamente para áreas críticas. Os usuários principais são os Agentes de Saúde, que realizam as vistorias, e os Gestores de Saúde, que monitoram as predições e equipes. Caso a inteligência preditiva falhe, o sistema perde seu diferencial e torna-se apenas um aplicativo comum de anotações.

### 1.2 Atributos de Qualidade (RNFs) Priorizados

•  Confiabilidade (Reliability): O modelo preditivo deve garantir uma precisão mínima de 85% na recomendação de áreas de risco, validada com focos reais encontrados em campo.
•  Desempenho (Performance): A recomendação da rota deve ser exibida na tela inicial no menor tempo possível, exigindo que a API retorne dados leves para não travar o dispositivo.
•  Disponibilidade (Resiliência): O sistema deve garantir que o trabalho do agente não pare em áreas sem conectividade 3G/4G, oferecendo suporte a operações offline.
•  Usabilidade: O aplicativo móvel precisa ser fácil de usar em ambiente externo e carregar rapidamente para atender à dinâmica de trabalho de campo.
•  Interoperabilidade: A API de IA deve ser capaz de retornar dados leves e compatíveis (JSON) para integração eficiente com o backend e o frontend mobile.

### 1.4 Classificação da Estratégia
-   **Classificação:** Ousada
-   **Justificativa:** A estratégia é classificada como ousada pois o diferencial do produto depende de um modelo de IA com alta meta de precisão (85%). Além disso, a arquitetura assume o risco tecnológico de implementar uma lógica complexa de sincronização e cache offline para garantir a funcionalidade em cenários urbanos de baixa conectividade..

## 1.5 Architecture Decision Record (ADR) Principal (Um Extra)
-   **Título:** ADR 001 – Adoção de Estratégia de Cache Offline para App Mobile. Status: Proposto.
-   **Contexto:** Agentes de saúde realizam vistorias em locais com instabilidade ou falta de rede 3G/4G. A dependência exclusiva de uma API online impediria o acesso às rotas e recomendações da IA em tempo real.
-   **Decisão:** Implementar armazenamento em cache local no dispositivo móvel para permitir que uma rota padrão e dados essenciais fiquem salvos no celular do agente.
-   **Justificativa e Trade-offs:**
•	Confiabilidade: Garante que o agente não fique travado na rua aguardando sinal de internet para trabalhar.
•	Performance: O carregamento da tela inicial torna-se muito mais rápido ao priorizar dados locais leves.
